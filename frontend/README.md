# PyAtlas Frontend

A beautiful, interactive visualization of the Python package ecosystem as a galaxy of interconnected packages.

## Overview

This frontend is built with:
- **Vite** - Fast build tool and dev server
- **React** - UI framework
- **TypeScript** - Type safety
- **Three.js** (@react-three/fiber, @react-three/drei) - 3D rendering
- **Tailwind CSS** - Styling
- **Zustand** - State management
- **Fuse.js** - Fuzzy search

## Getting Started

### Development

1. Install dependencies:
   ```bash
   npm install
   ```

2. Start the dev server:
   ```bash
   npm run dev
   ```

   The app will be available at `http://localhost:5173/pyatlas/`

### Building for Production

Build the static site:
```bash
npm run build
```

Preview the production build:
```bash
npm run preview
```

## Project Structure

```
src/
├── components/          # React components
│   ├── GalaxyCanvas.tsx    # Main Three.js canvas
│   ├── UIOverlay.tsx       # Overlay container
│   ├── SearchBar.tsx       # Search with autocomplete
│   ├── ClusterLegend.tsx   # Cluster filter UI
│   └── PackageDetail.tsx   # Selected package panel
├── types/              # TypeScript type definitions
│   └── index.ts
├── store/              # Zustand global state
│   └── useGalaxyStore.ts
├── utils/              # Utility functions
│   └── dataLoader.ts
├── App.tsx             # Main app component
└── main.tsx            # App entry point
```

## Data Files

The app expects two JSON files in `public/data/`:
- `packages.json` - Package data with coordinates and metadata
- `clusters.json` - Cluster metadata and labels

These files are generated by the Python preprocessing pipeline in the parent directory.

## Deployment

This app is configured for GitHub Pages deployment with the base path `/pyatlas/`.

To deploy:
1. Build the app: `npm run build`
2. Deploy the `dist/` directory to GitHub Pages
